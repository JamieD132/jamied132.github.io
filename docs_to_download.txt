        <h1>Darflen API Documentation</h1>

        <p class="text-gray-700 mb-6">
            This documentation provides information about all API endpoints found in Darflen's source code.
            If there are any missing endpoints or incorrect information, please let me know.
        </p>

        <div class="important-note">
            <strong>Note on Request Body Format:</strong>
            The server expects the body of all requests as <strong>formdata (<code>multipart/form-data</code>)</strong>.
            Also note that if there are any fields in a request that end in <code>[]</code>, you can provide multiple of that field.
        </div>

        <div class="note">
            <strong>Authentication:</strong>
            All API requests that require authentication need a token obtained from the login endpoint. This token must be provided as a Bearer token in the <code>Authorization</code> header, e.g., <code>Authorization: Bearer YOUR_AUTH_TOKEN</code>.
            Additionally, for POST requests, a CSRF token (retrieved from the <code>/data</code> endpoint or implicitly handled by session cookies) is expected to be sent as a form field named <code>csrf_token</code>.
        </div>
